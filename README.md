# ecommerce-analysis

# Анализ продаж за 2019 год (SQL-проект)

**Цель:** подготовка и анализ датасета с Kaggle (продажи по месяцам) с помощью PostgreSQL и DataGrip. Проект демонстрирует навыки очистки данных, объединения таблиц, преобразования типов и формулирования бизнес-выводов на основе SQL-запросов.

## Источник данных
Датасет [Sales Data](https://www.kaggle.com/datasets/...) (ссылка) — 12 CSV-файлов по месяцам 2019 года. Всего после очистки ~ 197907.

## Этапы работы
1. **Объединение таблиц**  
   - Проблема: разные типы данных в столбцах `Order ID`, `Quantity Ordered`, `Price Each`.  
   - Решение: приведение всех колонок к тексту в `UNION ALL`.  
   - Скрипт: [`sql/01_combine_tables.sql`](sql/01_combine_tables.sql)

2. **Очистка данных**  
   - Удаление строк-заголовков (где значения полей совпадали с именами колонок).  
   - Удаление пустых строк.  
   - Проверка корректности числовых полей с помощью регулярных выражений.  
   - Скрипт: [`sql/02_data_cleaning.sql`](sql/sql/02_data_cleaning.sql)

3. **Преобразование типов**  
   - Добавление числовых колонок `quantity_int` и `price_numeric`.  
   - Создание финальной таблицы `all_sales_2019_clean` с преобразованной датой и извлечённым городом из адреса.  
   - Скрипт: [`sql/03_type_conversion.sql`](sql/sql/03_type_conversion.sql) и [`sql/04_create_clean_table.sql`](sql/sql/04_create_clean_table.sql)

4. **Анализ и визуализация**  
   - Общая статистика (заказы, выручка, средний чек).  
   - Динамика выручки по месяцам (график).  
   - Топ-5 товаров и городов по выручке.  
   - Распределение заказов по дням недели.  
   - Скрипт: [`sql/05_analytical_queries.sql`](sql/sql/05_analytical_queries.sql)

## Как воспроизвести
1. Установите PostgreSQL и DataGrip.  
2. Скачайте датасет с Kaggle и импортируйте CSV-файлы в базу данных.  
3. Выполните скрипты в папке `sql/` в указанном порядке.  
4. Запустите аналитические запросы.
